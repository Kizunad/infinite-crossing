# 裁判硬规则清单 v0.1 (Judge Hard Rules v0.1)

> 本清单用于约束所有 LLM Agent（世界生成器 / 任务系统 / 旁白 / NPC），裁判拥有最终否决权。

## 一、总则（最高优先级）
**HR-0.1｜裁判至上**
- 裁判拥有对所有输出的最终裁决权
- 任一 Agent 输出如违反本清单，裁判必须修正或否决

**HR-0.2｜因果完整性不可破坏**
- 任一事件必须能追溯到明确原因
- 禁止“因为剧情需要”作为因果

**HR-0.3｜玩家输入具有最高行动优先权**
- 玩家明确行动意图不得被无理由忽略
- 系统不得以“你还没来得及做”为由跳过玩家行动

## 二、世界规则（World Rules）
### 2.1 物理层（Physics Layer）
**HR-2.1.1｜物理规则不可动态修改**
- 世界生成后，物理层规则在该世界内不可变
- 禁止新增/删除核心物理法则（如火药、魔法、电力）

**HR-2.1.2｜死亡必须符合物理可能性**
- 死亡事件必须符合物理层可实现路径
- 禁止“突然爆炸 / 无来源致死”

### 2.2 社会层（Society Layer）
**HR-2.2.1｜NPC 能力受身份约束**
- NPC 行为、资源、信息不得超出其社会身份合理范围
- 禁止“路人突然变成顶级战力”

**HR-2.2.2｜信息传播必须有来源**
- 任一情报必须能追溯到 NPC / 文献 / 观察
- 禁止系统直接灌输世界真相

### 2.3 叙事层（Narrative Layer）
**HR-2.3.1｜死亡必须可预防**
- 任一死亡事件必须存在至少一种合理规避路径
- 若无可规避路径 → 判定为违规死亡

**HR-2.3.2｜后果必须与选择成比例**
- 行动风险与结果严重度需匹配
- 禁止“低风险选择导致即时死亡”

## 三、任务系统规则（Quest System Rules）
**HR-3.1｜任务描述必须事实真实**
- 任务文本中所有陈述必须为“字面真实”
- 允许误导，但禁止直接谎言

**HR-3.2｜任务系统不得修改世界状态**
- 任务系统仅发布目标，不得生成事件
- 不得召唤 NPC / 灾难 / 物品

**HR-3.3｜坑只能来自信息，而非规则**
- 所有陷阱必须基于：
  - 信息缺失
  - 信息偏见
  - 认知误导
- 禁止规则级陷阱

## 四、回合与行动规则（Turn & Action）
**HR-4.1｜一回合一行动原则**
- 每回合玩家只能执行一个主行动
- 其余行为视为附属或失败

**HR-4.2｜裁判必须解析玩家意图**
- 自由输入必须被解析为：
  - 行动目标
  - 行动方式
  - 风险等级

**HR-4.3｜不可跳过关键决策点**
- 在高风险节点前，必须给予玩家选择权
- 禁止“自动推进到死亡”

## 五、数值与资源规则（Stats & Resources）
**HR-5.1｜数值变化必须有因果解释**
- 任一数值增减需对应具体事件
- 禁止隐性修改

**HR-5.2｜资源消耗优先于剧情便利**
- 若资源不足，行动必须失败或产生额外风险
- 禁止“剧情补给”

## 六、死亡与结算规则（Death & Settlement）
**HR-6.1｜死亡即世界封锁**
- 主角死亡后，该世界对其永久不可进入
- 世界状态永久存档

**HR-6.2｜死亡必须生成复盘报告**
- 复盘必须包含：直接死因、间接诱因、至少一个可避免路径

**HR-6.3｜结算奖励仅限一项**
- 玩家仅可带走一项（物品 / 能力 / 信息 / 人）
- 裁判需标注潜在副作用

## 七、信息与记忆规则（Memory & Knowledge）
**HR-7.1｜玩家仅拥有主观信息**
- 玩家界面不得显示未经历的事实
- 禁止上帝视角泄露

**HR-7.2｜世界档案馆为主观日志**
- 仅记录玩家经历与认知
- 不自动补全真相

## 八、输出格式强制规则（LLM Contract）
**HR-8.1｜回合输出结构不可变**
- 每回合输出必须包含：
  1. 世界状态变化
  2. 行动结果
  3. 数值变化
  4. 旁白叙事
  5. 下一步可选行动（≥3）

**HR-8.2｜旁白不得新增事实**
- 旁白仅文学化表达已裁定内容
- 禁止暗中推进剧情

## 九、违规处理
**HR-9.1｜违规优先修正而非合理化**
- 裁判应修正输出以符合规则
- 不得为违规行为事后找理由

**HR-9.2｜严重违规必须中断回合**
- 若无法修正，必须中止并提示系统错误
